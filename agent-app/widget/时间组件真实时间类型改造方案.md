# 时间组件真实时间类型改造方案

## 结论

日期时间字段统一使用 `datetime` 组件，前端/API raw value 固定为 `"YYYY-MM-DD HH:mm:ss"` 字符串；业务存储使用 `sdk/agent-app/types.Time`，数据库列使用真实时间类型，例如 `gorm:"type:datetime"`，不把时间落成数字时间戳或普通字符串。

不再保留其他日期时间组件协议。提示词、SDK、前端和案例都只围绕 `datetime + types.Time` 生成和解析。

## 目标

- 前端渲染、表单提交、表格筛选、工作台工具调用都使用人可读时间字符串。
- 数据库保留真实时间语义，范围查询、排序、索引和 BI 分析都走数据库时间列能力。
- 大模型不需要手写数字时间，只使用 SQL 风格白名单表达式。
- `render_default` 只表示前端渲染默认值，不等同于数据库默认值。

## SDK 约定

字段示例：

```go
CreatedAt types.Time `json:"created_at" gorm:"column:created_at;type:datetime;autoCreateTime" widget:"name:创建时间;type:datetime;format:YYYY-MM-DD HH:mm:ss" hide:"create,update"`
Deadline  types.Time `json:"deadline" gorm:"column:deadline;type:datetime" widget:"name:截止时间;type:datetime;format:YYYY-MM-DD HH:mm:ss;render_default:DATE_ADD(CURRENT_DATE, INTERVAL 1 DAY)"`
```

`DateTime` 配置：

```go
type DateTime struct {
    Format        string `json:"format,omitempty"`
    Disabled      bool   `json:"disabled,omitempty"`
    RenderDefault string `json:"render_default,omitempty"`
}
```

`render_default` 支持 SQL 风格白名单表达式：

| 表达式 | 说明 |
|--------|------|
| `CURRENT_TIMESTAMP` | 当前时间 |
| `CURRENT_DATE` | 今天零点 |
| `DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 7 DAY)` | 当前时间往前 7 天 |
| `DATE_ADD(CURRENT_TIMESTAMP, INTERVAL 1 HOUR)` | 当前时间后一小时 |
| `DATE_ADD(CURRENT_DATE, INTERVAL 1 DAY)` | 明天零点 |

只支持白名单表达式，不执行任意 SQL。

## types.Time

`sdk/agent-app/types.Time` 负责统一 Go、JSON 和数据库协议：

- `MarshalJSON` 输出 `"2006-01-02 15:04:05"`。
- `UnmarshalJSON` 支持 `"2006-01-02 15:04:05"`、`"2006-01-02"`、RFC3339、空字符串和 `null`。
- `Scan` 支持数据库返回 `time.Time`、`string`、`[]byte`、`nil`。
- `Value` 返回 `time.Time` 或 `nil`，让数据库驱动写入真实时间类型。
- `String()` 返回 `2006-01-02 15:04:05`。

Go layout 和前端 format 不是同一套语法：Go 使用 `2006-01-02 15:04:05`，前端使用 `YYYY-MM-DD HH:mm:ss`。

## 前端约定

`DateTimeWidget`：

- 编辑态使用日期时间选择器。
- 提交 raw value 为 `"YYYY-MM-DD HH:mm:ss"`。
- 详情、表格、响应页统一格式化展示。
- 范围搜索生成 `gte/lte` 字符串参数。
- 从 URL 恢复搜索条件时保持字符串，不做数字转换。

## 工作台查询

`run_table_search` 的时间范围直接使用字符串或 SQL 风格白名单表达式：

```text
gte=created_at:DATE_SUB(CURRENT_TIMESTAMP, INTERVAL 7 DAY)&lte=created_at:CURRENT_TIMESTAMP
```

工具执行前解析为：

```text
gte=created_at:2026-04-14 16:30:00&lte=created_at:2026-04-21 16:30:00
```

## 验收标准

- SDK 不再输出其他日期时间组件类型。
- 前端只注册 `datetime` 日期时间组件。
- 搜索、URL 恢复、表单默认值都按字符串时间协议处理。
- `CURRENT_TIMESTAMP`、`CURRENT_DATE`、`DATE_ADD/DATE_SUB(..., INTERVAL n UNIT)` 能被正确解析。
- 文档和提示词只生成 `datetime + types.Time`。
